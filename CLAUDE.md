# Khaata

A personal-finance / expense-tracking mobile app. It reads bank & UPI **SMS
alerts** off the device, parses them into categorized debit transactions, and
presents spending, budgets, and recent activity. When SMS isn't available
(iOS, web, or Expo Go) it falls back to a built-in demo dataset so the whole
UI stays usable.

## Tech stack

- **Expo SDK 52** (managed workflow) + **React Native 0.76.5** (new architecture enabled)
- **expo-router 4** for file-based navigation, **TypeScript**, typed routes
- Custom local Expo native module **`sms-reader`** (Android-only, Kotlin)
- Plus Jakarta Sans fonts, `react-native-svg` for the donut chart

## Project layout

```
app/                       expo-router routes (file-based)
  index.tsx                splash / entry redirect
  onboarding.tsx           connect-SMS onboarding
  (tabs)/                  bottom-tab screens: index (home), activity, spending, budgets
  transaction/[id].tsx     transaction detail (modal)
src/
  context.tsx              AppProvider — global state, connectSms(), recategorize()
  data.ts                  Transaction type + INITIAL_TXNS demo dataset
  sms.ts                   importSmsTransactions(): permission + read + parse orchestration
  smsParser.ts             SMS → Transaction parsing (amount/merchant/category/debit-vs-credit)
  theme.ts                 colors, spacing, category metadata
components/                Icon.tsx, DonutChart.tsx
modules/sms-reader/        custom native module
  index.ts                 JS API: readInbox(), isSmsReadingSupported (requireOptionalNativeModule)
  android/.../SmsReaderModule.kt   Kotlin: reads the SMS inbox via ContentResolver
android/                   generated by `expo prebuild` — DO NOT hand-edit; regenerate instead
```

`scripts/testParser.ts` exercises `smsParser.ts` against sample messages.

## How the SMS pipeline works

`connectSms()` (in [src/context.tsx](src/context.tsx)) → `importSmsTransactions()`
([src/sms.ts](src/sms.ts)) requests the `READ_SMS` permission and calls the native
`readInbox()`, then `smsParser.ts` turns raw messages into `Transaction`s:
debit keywords select spends (credits/OTPs/promos are ignored), a rupee amount
is regex-extracted, a merchant is guessed from "to/at/towards …", and the
merchant+body are matched against `CATEGORY_KEYWORDS` to assign a category.
`isSmsReadingSupported` is false off-device, so the app silently keeps demo data.

## Running locally

```bash
npm install
npx expo start            # press w (web), or scan QR with Expo Go
npx expo start --web      # web only
```

Note: the `sms-reader` native module is **not** in Expo Go — real SMS reading
only works in a dev build or the installed APK. In Expo Go / web the app runs
on demo data.

## Building the Android APK (local)

This machine's toolchain is set up via Homebrew. Required because the system
default JDK is too new for Android Gradle:

- **JDK 17** — `/opt/homebrew/opt/openjdk@17` (Homebrew `openjdk@17` formula).
  Do NOT use the system JDK 26; the Android Gradle Plugin rejects it.
- **Android SDK** — `/opt/homebrew/share/android-commandlinetools`
  (`brew install --cask android-commandlinetools`), with platform-tools,
  `platforms;android-35`, `build-tools;35.0.0`, NDK, and cmake installed.

Env vars live in [.android-env.sh](.android-env.sh) — `source` it before any
Gradle/SDK command:

```bash
source .android-env.sh                 # sets JAVA_HOME, ANDROID_HOME, PATH
npx expo prebuild --platform android    # (re)generate android/ — only if missing/stale
cd android && ./gradlew assembleRelease --no-daemon
```

Output APK: `android/app/build/outputs/apk/release/app-release.apk`.

The release build is **signed with the debug keystore** (`android/app/debug.keystore`,
configured in `android/app/build.gradle`), so the APK installs and runs but is
NOT Play-Store-ready. For a real release, generate a keystore and add a proper
`release` signingConfig. For a debug build instead, use `./gradlew assembleDebug`.

### Gotchas

- `expo prebuild` regenerates `android/`; never hand-edit files there — change
  config in `app.json` / module configs and re-run prebuild.
- The Homebrew Temurin **cask** (`temurin@17`) fails to install headless (its
  `.pkg` needs an admin password via terminal). Use the **formula** `openjdk@17`.
- Gradle may auto-download extra NDK/build-tools versions pinned by
  dependencies (e.g. NDK 26.1, build-tools 34) — this is expected.
- `READ_SMS` is Android-only and declared in `app.json`. Reading SMS at runtime
  requires the user to grant the permission on the device.
- **Kotlin/Compose version pin** (`android/gradle.properties` → `android.kotlinVersion=1.9.24`):
  RN 0.76's Gradle plugin pins the actual Kotlin compiler to 1.9.24, but the
  template's `ext.kotlinVersion` defaults to 1.9.25, which makes expo-modules-core
  select Compose Compiler 1.5.15 (requires 1.9.25) → `compileReleaseKotlin` fails.
  Pinning to 1.9.24 makes it pick the matching Compose Compiler 1.5.14. Since
  `gradle.properties` is regenerated by prebuild, re-add this after any prebuild
  (or move it into an `expo-build-properties` config in `app.json` to persist it).
